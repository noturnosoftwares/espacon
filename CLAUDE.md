# HelpDesk

## Fonte de Regras

Este projeto utiliza como base arquitetural o repositório corporativo:

noturno-web-template

Antes de qualquer implementação, revisão, refatoração ou geração de código, devem ser consideradas as regras definidas no template corporativo.

---

# Ordem de Prioridade

As decisões devem seguir a seguinte ordem:

1. Regras deste projeto (HelpDesk)
2. Regras do noturno-web-template
3. Boas práticas amplamente adotadas pelo mercado
4. Documentação oficial das tecnologias utilizadas

---

# Papel da IA

A IA atua como arquiteto assistente e executor técnico.

A IA não deve impor arquiteturas próprias quando existirem padrões definidos pela Noturno Softwares.

A IA deve adaptar-se ao padrão existente.

---

# Antes de Implementar

Antes de criar qualquer código:

* Ler CLAUDE.md
* Ler docs/app
* Ler docs/architecture
* Ler docs/ui
* Ler docs/specifications do módulo envolvido

Se a documentação não existir ou estiver incompleta, solicitar a documentação antes da implementação.

---

# Regra Fundamental

Nenhuma implementação deve ser iniciada apenas com base em suposições.

A documentação do projeto possui prioridade sobre decisões automáticas da IA.

---

# Escopo

Este projeto representa o portal de gestão da Noturno Softwares.

Todas as implementações devem considerar:

* Franquias
* Representantes
* Multiempresa
* Crescimento futuro
* Reaproveitamento
* Escalabilidade
* Manutenibilidade
* Compatibilidade futura

Evitar decisões que exijam grandes refatorações para expansão futura.
